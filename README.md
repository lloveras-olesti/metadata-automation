# auto-metadata

Automated SEO metadata audit and implementation pipeline for e-commerce websites.

Takes Screaming Frog crawl exports as input, generates corrected title tags and meta descriptions using deterministic rules and Claude AI, and writes them directly to the CMS via API — no manual copy-paste, no backend access required.

**Stack:** Python · Docker · Pandas · Claude API (Anthropic) · PrestaShop Webservice API

---

## What it does

1. **Ingests** Screaming Frog CSV exports (missing titles, missing metas, length violations)
2. **Classifies** every URL by type (product, category, CMS page) and language
3. **Fixes** title length issues deterministically (trim dash segments, extend with brand suffix)
4. **Generates** AI-optimised metadata for high-priority pages (categories) via Claude API
5. **Validates** all generated values against character limits before writing
6. **Writes** directly to the live PrestaShop store via Webservice API

Everything is configured in a single `config.yaml` file — brand name, keywords, CTAs, language settings, AI model. No values are hardcoded in the scripts.

---

## Case study: Llibres del Mirall (Barcelona)

This pipeline was built and executed for [Llibres del Mirall](https://www.llibresdelmirall.com), an antiquarian bookshop in Barcelona with a PrestaShop catalogue of ~7,000 URLs in Spanish and Catalan.

**Results:**
- 7,009 URLs audited and processed
- ~5,890 product title tags corrected (trim + AI generation for edge cases)
- ~904 category pages given SEO-optimised meta titles and descriptions via Claude API
- Dry-run passed with 6,982 OK and 26 expected errors (deleted products, module-specific pages)
- Zero manual copy-paste — all changes written directly to production via API

**Blockers encountered:** The `blocklayered` (faceted navigation) module was hooked into PrestaShop's `actionCategoryUpdate` event and caused HTTP 400 errors on all PUT requests. Fix: temporarily disable the module, run the live write, re-enable and re-index. Documented in the write script's error log.

## Setup

### 1. Configure for your client

Edit `config.yaml` with the client's details:

```yaml
client:
  name: "Your Client Name"
  base_domain: "https://www.clientsite.com"
  description: >
    A short description of the business for AI prompts.
  languages: [es, ca]
  language_names:
    es: "Spanish"
    ca: "Catalan"

brand:
  suffix_short: " - Your Client Name"
  suffix_long:
    es: " - Your Client Name, Descriptor in Spanish"
    ca: " - Your Client Name, Descriptor in Catalan"
```

See `config.yaml` for the full reference with all available parameters.

### 2. Set environment variables

```bash
cp .env.example .env
# Edit .env with real API credentials
```

### 3. Build the Docker image

```bash
docker-compose build pipeline
```

### 4. Verify API access (PrestaShop)

```bash
docker-compose run --rm pipeline python scripts/verify_prestashop.py
```

This prints the language IDs you need to set in `.env`.

## Running the pipeline

All scripts run inside Docker. Always run from the project root.

```bash
# Step 0: parse site nav HTML → category lookup CSVs
# First: copy menu HTML files to data/menus/menu_es.html, menu_ca.html
docker-compose run --rm pipeline python scripts/build_category_lookup.py

# Step 1: ingest Screaming Frog CSVs → master_urls.csv
# First: copy SF exports to data/screaming-frog/
docker-compose run --rm pipeline python scripts/classify_urls.py

# Step 2: fetch missing product titles from the API
docker-compose run --rm pipeline python scripts/fetch_missing_titles.py

# Step 3: fix title/meta length issues deterministically
docker-compose run --rm pipeline python scripts/fix_lengths.py

# Step 3b: AI titles for products that couldn't be trimmed mechanically
docker-compose run --rm pipeline python scripts/generate_ai_titles.py

# Step 4: generate missing meta descriptions (products and geo pages)
docker-compose run --rm pipeline python scripts/generate_missing_meta.py

# Step 4b: AI-optimised meta titles + descriptions for category pages
docker-compose run --rm pipeline python scripts/generate_category_meta.py --dry-run
docker-compose run --rm pipeline python scripts/generate_category_meta.py

# Step 5: validate all generated values
docker-compose run --rm pipeline python scripts/pilot_validation.py

# Step 6: write to production
docker-compose run --rm pipeline python scripts/write_to_prestashop.py --dry-run
docker-compose run --rm pipeline python scripts/write_to_prestashop.py --live
```

## Screaming Frog input files

Place these standard Screaming Frog exports in `data/screaming-frog/` before running Script 1:

| File | Issue tracked |
|------|--------------|
| `page_titles_missing.csv` | Pages with no title tag |
| `page_titles_below_30_characters.csv` | Title tags too short |
| `page_titles_over_60_characters.csv` | Title tags too long |
| `meta_description_missing.csv` | Pages with no meta description |
| `meta_description_below_70_characters.csv` | Meta descriptions too short |
| `meta_description_over_155_characters.csv` | Meta descriptions too long |

Export each from Screaming Frog: **Bulk Export → Page Titles / Meta Description → [filter]**

## Adapting for a different CMS

The audit and generation scripts (`classify_urls.py` through `pilot_validation.py`) are CMS-agnostic. Only two parts are PrestaShop-specific:

1. **`scripts/utils.py` → `classify_url()`**: URL pattern matching (`/{id}-{slug}.html` for products, `/{id}-{slug}` for categories). Adapt the regex patterns for your CMS's URL structure.

2. **`scripts/write_to_prestashop.py`**: The writer uses PrestaShop's XML Webservice with Basic Auth. For WordPress, implement an equivalent using the WordPress REST API with Application Passwords.

3. **`scripts/fetch_missing_titles.py`**: Fetches product names from PrestaShop's `/api/products/{id}` endpoint. For other CMS platforms, adapt or skip this script if the Screaming Frog export already contains the title data.

## Configuration reference

All parameters in `config.yaml`:

| Key | Description |
|-----|-------------|
| `client.name` | Brand name — used in prompts and suffixes |
| `client.base_domain` | Full site URL |
| `client.description` | Business description for AI prompts |
| `client.languages` | ISO codes used in URL paths |
| `client.language_names` | Human-readable names for AI prompts |
| `brand.suffix_short` | Short suffix appended to titles if it fits |
| `brand.suffix_long` | Long suffix for padding short titles (per language) |
| `meta_templates.prefix/infix` | Deterministic meta description template parts |
| `seo_limits.title_min/max` | Character limits for title tags (default: 30–60) |
| `seo_limits.meta_min/max` | Character limits for meta descriptions (default: 70–155) |
| `ai.model` | Claude model ID for AI generation |
| `menu.category_menu_id` | HTML `id` of the category nav `<ul>` |
| `menu.geo_menu_id` | HTML `id` of the geo nav `<ul>` (null if not applicable) |
| `menu.category_root_skip_patterns` | URL substrings identifying the root catalog link |
| `seo_generation.relevant_page_type` | Page type to target with AI generation (e.g. "category") |
| `seo_generation.keyword_pool` | SEO keywords by language for AI prompts |
| `seo_generation.ctas` | Call-to-action phrases by language |
| `seo_generation.include_brand_in_meta_description` | Whether AI may include brand in meta desc |
